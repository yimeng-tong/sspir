# Colab Pro 资源需求清单

生成时间：2026-03-27

## 1. 结论先说

如果你有 Colab Pro，本课题现阶段最现实的策略不是“从零完整训练所有模型”，而是：

1. 优先完成官方权重推理复现。
2. 在此基础上做小规模消融和局部改动验证。
3. 把完整训练留作可选增强项，而不是论文能否完成的前置条件。

原因很简单：你的方法同时涉及 Retinex 分解、MLLM 提示生成和扩散恢复，完整链路对显存、存储和运行时长都有明显要求，而 Colab Pro 更适合做推理、轻量复现和小规模训练。

## 2. 你现在最需要的资源

### 必需资源

- Google Drive 可用空间：建议至少 `80GB`
- 单卡 GPU 运行环境：可用于 PyTorch + diffusers
- 稳定网络：用于下载预训练模型、数据集和依赖

### 理想 GPU 条件

- 更适合：`L4` 或更高规格 GPU
- 可以接受：`T4`

经验判断：

- `T4` 足够做大多数基线推理与部分小规模消融。
- 如果要跑 Stable Diffusion 类方法并保留较舒适的 batch / resolution，`L4` 会明显更稳。

## 3. 建议下载的数据与权重

### 数据集

- `LOL-v1`
- `LOLv2-Real`
- `LOLv2-Synthetic`
- 可选真实展示集：`LIME`、`NPE`、`DICM`、`MEF`、`VV`

### 主要模型权重

- `Stable Diffusion v1.5`
- `SSP-IR` 官方权重
- `Reti-Diff` 官方权重
- `Diff-Retinex` 官方权重
- `URetinex-Net` 官方权重
- 可选：`EnlightenGAN`、`ILR-Net` 公开权重

### 语义分支资源

- `CLIP` 权重
- 如需离线提示生成：一个可用的 MLLM 推理入口

建议：

- 不要在 Colab 上直接部署超重的本地 MLLM 训练。
- 更现实的做法是离线生成提示词，或使用外部 API/现成推理入口生成固定提示文件。

## 4. 我建议你在 Colab 上优先做的事

### 第一优先级：可写进论文的结果

- 跑通 `URetinex-Net`、`Diff-Retinex`、`Reti-Diff` 的官方推理
- 统一整理输出图像
- 计算 `PSNR / SSIM / LPIPS / NIQE`
- 形成第四章主表

### 第二优先级：与你方法直接相关的验证

- 做 `R` 图替代原图的提示生成对比
- 做 `SNR-aware mask` 与普通 FFT 约束的对比
- 做小规模样本消融

### 第三优先级：完整训练或微调

- 若资源允许，再尝试冻结扩散主干下的小规模训练
- 不建议一开始就完整训练整套多模块系统

## 5. 对 Colab Pro 的现实判断

Colab Pro 足以支持：

- 官方模型推理复现
- 小规模 paired 数据集验证
- 少量消融实验
- 第四章主体结果的搭建

Colab Pro 不一定适合：

- 长时间稳定完整训练多套扩散模型
- 大规模本地 MLLM 联合训练
- 高分辨率、长步数、多模型并行的全流程实验

## 6. 我下一步最希望你能提供的资源

如果你准备开始跑实验，最有价值的不是再给我更多论文，而是下面这些：

1. 一个你愿意优先跑的实验平台：
   - Colab Pro
   - 本地机器
2. 你当前已经下载好的数据集和权重清单。
3. 你是否接受“先用官方权重做结果，再做局部消融”的实验策略。

## 7. 我的建议

就你现在的阶段，最划算的做法是：

- 用 Colab Pro 先拿下 `URetinex-Net + Diff-Retinex + Reti-Diff + SSP-IR` 的推理结果；
- 同时把第四章写到可以直接填表的程度；
- 再决定是否做你自己的小规模实现与消融。

这条路径最稳，也最符合本科论文的时间收益比。
